Service rules for a Discord birthday bot built around scoped data and practical admin control.

These terms explain the responsibilities, limits, and acceptable use rules that apply when you install or use Bdayblaze in a Discord server. They are written for the actual bot surface: birthday records, celebration flows, private admin tooling, media validation, and lightweight hosted operation.

Discord-first scope

Bdayblaze is a Discord-native celebration bot, not a guaranteed delivery service, archive, or enterprise platform.

Admin accountability

Server admins remain responsible for routing, permissions, exports, and the custom content they configure.

Custom content limits

Templates, wishes, links, and media stay the server's responsibility even when Bdayblaze validates or delivers them.

No uptime guarantee

Discord, hosting, storage, and external media sources can affect scheduling, previews, imports, exports, and delivery.

Service scope

Bdayblaze provides Discord-native birthday features such as birthday record storage, server-scoped visibility controls, Celebration Studio configuration, private setup and preview flows, timeline and quest state, compact surprise tracking, anniversary support, analytics, health checks, and import/export tooling.

Bdayblaze is offered as a lightweight bot service for normal server operations. It is not an enterprise workflow platform, backup service, legal archive, or paid SLA product.
