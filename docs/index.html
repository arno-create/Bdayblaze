Standout features

Compact features with real product weight

Birthday Capsules

Friends queue wishes ahead of time. Wishes stay hidden until the member birthday window opens.

Birthday Quests

Optional quest goals track revealed wishes, shared announcement reactions, and an optional birthday check-in without broad activity monitoring.

Birthday Surprises

Compact, weighted reward selection with manual-only Nitro concierge follow-up and clear admin audit state.

Timeline profiles

/birthday timeline shows countdowns, active celebration state, wishes, quest progress, and compact history.

Media Tools

Direct media, webpage wrappers, unsupported files, and unsafe URLs are surfaced explicitly so admins know why a save was accepted or blocked.

Admin analytics

Lightweight command-based analytics surface birthdays stored, wishes queued, quest completions, surprise counts, and recovery issues without a dashboard backend.

How it works

Discord-native, restart-safe, and free-tier conscious

1

Members save birthdays privately

Month and day are required, birth year is optional, and visibility stays guild-scoped.

2

Admins configure Studio

Routing, visuals, safety checks, and previews stay private to the operator.

3

Scheduler claims durable work

Announcements, DMs, capsule reveals, and role windows run from persisted event rows.

4

Reaction goals refresh live

Shared birthday posts can drive quest reactions through debounced refreshes only.

Privacy

Personal data handled with restraint

  • Birthdays stay scoped to the current server and are never shared cross-server.
  • Birth year is optional and hidden by default.
  • Unrevealed wishes stay private until the celebration day.
  • Reaction quests store only celebration-level totals tied to the announcement post.
  • Deletion paths remain available through member and admin remove flows.

What Bdayblaze does not do

  • No Message Content intent
  • No Jinja, eval, or arbitrary attribute access
  • No automated Nitro gifting or payment flow
  • No AI-generated cake or card system in this release

Reliability

Built for constrained hosting

Single-process async bot, Postgres-backed scheduling, bounded recovery work, and no extra worker stack.

Operator clarity

Diagnostics without a dashboard

Health endpoints, command-based analytics, and Studio diagnostics keep operators informed without widening the architecture.

Static deployment

One clear site entrypoint

The public site is the static docs/ bundle. The bot runtime and the site are intentionally deployed separately.
