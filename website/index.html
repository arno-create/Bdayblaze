Standout features

Compact features with real product weight

Birthday Capsules

Friends queue wishes ahead of time. Wishes stay hidden until reveal and remain private when a public birthday route is unavailable.

Birthday Quests

Optional low-noise quest goals track revealed wishes, shared-post reactions, and an optional birthday check-in without broad activity monitoring.

Birthday Surprises

Weighted surprise rewards stay safe and auditable, including manual-only Nitro concierge records with fulfillment tracking.

Timeline Profiles

`/birthday timeline` gives members a clean celebration card with countdowns, streaks, wishes, surprises, and compact history.

Media Tools

Direct media, webpage wrappers, unsupported assets, and unsafe links are explained clearly so admins know exactly why a URL was accepted or blocked.

Admin analytics

Lightweight command-based analytics surface stored birthdays, queued wishes, quest completions, surprise counts, and recent recovery issues without a dashboard backend.

How it works

Discord-native, restart-safe, and low cost by design

1

Members save their birthday privately

Birthdays are stored per server membership with optional birth year and visibility controls.

2

Admins configure Studio and Media Tools

Routing, visuals, safety checks, and previews stay private to the operator using slash commands.

3

The scheduler claims durable work

Announcement batches, capsule reveals, and role flows run from persisted records with bounded recovery work after restarts.

4

Quest reactions refresh live without extra workers

Shared birthday announcement posts can drive quest reaction goals through debounced reaction refreshes and a small tracked-message cache.

Canonical commands

One coherent command surface

Members

  • `/birthday set`, `view`, `remove`, `privacy`
  • `/birthday today`, `next`, `upcoming`, `month`, `twins`, `list`
  • `/birthday wish add`, `list`, `remove`
  • `/birthday capsule preview`
  • `/birthday quest status`, `check-in`
  • `/birthday timeline`

Admins

  • `/birthday studio`, `setup`, `test-message`, `analytics`, `health`
  • `/birthday surprise queue`, `fulfill`
  • `/birthday member view`, `set`, `remove`
  • `/birthday anniversary settings`, `sync`
  • `/birthday event add`, `edit`, `list`, `remove`
  • `/birthday import`, `export`

Privacy

Personal data handled with restraint

  • Birthdays stay scoped to the current server, never shared cross-server.
  • Birth year is optional and hidden by default.
  • Unrevealed wishes stay private until the celebration day.
  • Reaction quests store only celebration-level totals tied to the shared birthday post.
  • Deletion paths remain available through member and admin remove flows.

What Bdayblaze does not do

  • No Message Content intent
  • No Jinja, eval, or arbitrary attribute access
  • No automated Nitro gifting or payment flow
  • No AI-generated cake/card system in this release

Trust and reliability
