The product already looks like something admins can trust on day one.

Bdayblaze is strongest when its public moments and private controls feel equally intentional. The core release surfaces are already there in Discord.

01

Celebration Studio keeps the whole birthday experience under one roof.

Birthday announcements, DMs, anniversaries, shared visuals, capsules, quests, surprises, and yearly events all stay inside a single private admin flow.

  • Shared visuals and previews stay close to the actual celebration copy.
  • Reset paths are explicit, so mistakes do not become hidden state.
  • Admins can tune the release feel without leaving Discord.

02

Birthday Setup handles routing, eligibility, and quiet operational details.

The setup flow is built for real server administration: channel routing, timezones, membership rules, role toggles, anniversary handling, and safety settings before anything goes public.

  • Birthday, anniversary, and DM behavior are visible from one panel.
  • Timezone defaults and role rules are part of the flow, not hidden config.
  • Studio safety and operator previews stay close to setup.

03

Media Tools protects the final post before a bad link ever reaches your server.

Admins get clear feedback for direct media, webpage wrappers, unsupported files, and unsafe URLs. Failed saves do not silently wipe the last working media entry.

  • Validation is explicit instead of a vague pass or fail.
  • Signed URLs and extensionless object links can still work when validation confirms them.
  • Bad media stays an admin problem, not a broken public post.

How it works

Simple enough for members, controlled enough for admins.

  1. 1

    Members save birthdays once

    Month and day are required. Birth year is optional. Visibility stays scoped to the current server.

  2. 2

    Admins shape the experience privately

    Studio, setup, previews, safety rules, analytics, and health checks remain operator-facing.

  3. 3

    Bdayblaze schedules from real member time

    Announcements and celebration windows use each member's effective timezone and persisted event records.

  4. 4

    The server gets a cleaner celebration moment

    Capsules unlock, quests progress, surprises stay compact, and recovery remains bounded after restarts.

Privacy and trust

Careful data handling without burying the product in legal fog.

Bdayblaze keeps birthdays scoped to the current server, avoids Message Content intent, stores only compact quest and timeline state, and keeps admin tooling private by default. For the full details, read the public privacy policy.

Stored per server

Birthday month/day, optional birth year, optional timezone override, and a server-scoped visibility choice.

Private where it matters

Birthday wishes reveal on the birthday window, and admin setup, Studio, previews, analytics, health, and import/export flows stay private to admins.

Minimal tracking

Reaction quests use celebration-level totals tied to the shared announcement post. No Message Content intent, no global profile, no automated Nitro gifting flow.
